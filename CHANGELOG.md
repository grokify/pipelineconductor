## [v0.1.0] - 2026-01-28

### Highlights

- Multi-org GitHub repository scanning with Cedar policy-as-code evaluation
- Four output formats (JSON, Markdown, SARIF, CSV) with GitHub Security integration
- Built-in rate limiting with exponential backoff for large-scale scans

### Added

- CLI with `scan`, `validate`, and `version` commands using Cobra framework
- GitHub collector for repository, workflow, and branch protection data
- Cedar policy engine with support for custom and built-in policies
- Built-in policies: `require-workflow`, `require-tests`, `go-versions`
- Profile system with three built-in profiles: `default`, `modern`, `legacy`
- JSON report format with compliance summary and per-repo details
- Markdown report format with status icons and severity indicators
- SARIF 2.1.0 report format for GitHub Security tab integration
- CSV report format for spreadsheet analysis
- Repository filtering by language, topics, visibility, archived/fork status
- Policy validation command with syntax checking
- Rate limiting with automatic retry, exponential backoff, and GitHub `X-RateLimit-*` header support
- Example Cedar policies for Go projects (merge, versions, matrix, dependencies, reusable-workflow)

### Dependencies

- `github.com/cedar-policy/cedar-go` v1.4.0 for policy evaluation
- `github.com/google/go-github/v82` for GitHub API
- `github.com/grokify/mogo` for HTTP retry transport
- `github.com/spf13/cobra` for CLI framework
- `github.com/spf13/viper` for configuration management

### Documentation

- MkDocs documentation site with Material theme (21 pages)
- CLI reference for scan, validate, and configuration
- Cedar policy syntax guide and writing policies tutorial
- Profile system documentation with customization guide
- GitHub Actions integration guide with workflow examples
- SARIF integration guide for GitHub Security tab
- Contributing guide and development roadmap
